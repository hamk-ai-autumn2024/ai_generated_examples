Flask Tutorial: Learn Python Web Development

Flask is a lightweight web framework for Python that provides essential tools and features for building web applications. It's designed to make getting started quick and easy, with the ability to scale up to complex applications.

Introduction to Flask

What is Flask?

Flask is a micro web framework written in Python. It's classified as a microframework because it does not require particular tools or libraries. It has no database abstraction layer, form validation, or any other components where pre-existing third-party libraries provide common functions.

Why Use Flask?

  • Lightweight and Flexible: Flask doesn't enforce specific project structure or dependencies
  • Easy to Learn: Simple syntax and minimal setup requirements
  • Extensible: Large ecosystem of extensions for added functionality
  • Built-in Development Server: No need for external servers during development
  • Integrated Unit Testing Support: Built-in support for unit testing

Flask vs Django

Flask

  • Microframework
  • Minimal dependencies
  • Flexible structure
  • Manual setup required
  • Good for small to medium apps

Django

  • Full-featured framework
  • Batteries included
  • Opinionated structure
  • Built-in admin panel
  • Good for large, complex apps

Installing Flask

Before we start building applications with Flask, we need to install it. Flask requires Python 3.7 or newer.

Using pip

The easiest way to install Flask is using pip:

pip install Flask

Virtual Environment (Recommended)

It's best practice to use a virtual environment for Python projects:

# Create a virtual environment python -m venv myenv # Activate the virtual environment # On Windows: myenv\Scripts\activate # On macOS/Linux: source myenv/bin/activate # Install Flask pip install Flask

Your First Flask Application

Let's create a simple "Hello, World!" application to get started with Flask.

Basic Flask App

Create a file called app.py:

from flask import Flask app = Flask(__name__) @app.route('/') def hello(): return 'Hello, World!' if __name__ == '__main__': app.run(debug=True)

Running the Application

Save the file and run it:

python app.py

You should see output similar to:

* Running on http://127.0.0.1:5000 * Debug mode: on

Open your web browser and go to http://127.0.0.1:5000 to see your application.

Routing in Flask

Routing refers to mapping URLs to functions in your application.

Basic Routes

from flask import Flask app = Flask(__name__) @app.route('/') def home(): return 'Home Page' @app.route('/about') def about(): return 'About Page' @app.route('/contact') def contact(): return 'Contact Page'

Dynamic Routes

You can create dynamic routes with variable parts:

@app.route('/user/<username>') def show_user(username): return f'User: {username}' @app.route('/post/<int:post_id>') def show_post(post_id): return f'Post ID: {post_id}'

HTTP Methods

Specify which HTTP methods a route should accept:

@app.route('/login', methods=['GET', 'POST']) def login(): if request.method == 'POST': return 'Processing login...' else: return 'Login form'

Templates in Flask

Templates allow you to separate HTML from Python code and make your applications more maintainable.

Setting Up Templates

Create a templates directory in your project folder:

project/ ├── app.py └── templates/ ├── base.html ├── home.html └── about.html

Basic Template

templates/base.html:

<!DOCTYPE html> <html> <head> <title>{% block title %}My Flask App{% endblock %}</title> </head> <body> <nav> <a href="{{ url_for('home') }}">Home</a> <a href="{{ url_for('about') }}">About</a> </nav> {% block content %}{% endblock %} </body> </html>

templates/home.html:

{% extends "base.html" %} {% block title %}Home - My Flask App{% endblock %} {% block content %} <h1>Welcome to My Flask App</h1> <p>This is the home page.</p> {% endblock %}

Rendering Templates

from flask import Flask, render_template app = Flask(__name__) @app.route('/') def home(): return render_template('home.html') @app.route('/about') def about(): return render_template('about.html', title='About')

Template Variables

@app.route('/user/<name>') def user(name): return render_template('user.html', name=name)

templates/user.html:

{% extends "base.html" %} {% block content %} <h1>Hello, {{ name }}!</h1> <p>Welcome to our application.</p> {% endblock %}

Working with Forms

Flask makes it easy to handle form data from HTML forms.

HTML Form

templates/contact.html:

{% extends "base.html" %} {% block content %} <h1>Contact Us</h1> <form method="POST"> <div> <label for="name">Name:</label> <input type="text" id="name" name="name" required> </div> <div> <label for="email">Email:</label> <input type="email" id="email" name="email" required> </div> <div> <label for="message">Message:</label> <textarea id="message" name="message" required></textarea> </div> <button type="submit">Send</button> </form> {% endblock %}

Processing Form Data

from flask import Flask, render_template, request, redirect, url_for app = Flask(__name__) @app.route('/contact', methods=['GET', 'POST']) def contact(): if request.method == 'POST': name = request.form['name'] email = request.form['email'] message = request.form['message'] # Process the form data (e.g., save to database, send email) print(f"Name: {name}, Email: {email}, Message: {message}") # Redirect to a thank you page return redirect(url_for('thank_you')) return render_template('contact.html') @app.route('/thank-you') def thank_you(): return render_template('thank_you.html')

Database Integration

Flask works well with various databases. Here's how to integrate SQLite with Flask.

Using SQLite with Flask

import sqlite3 from flask import Flask, render_template, request, redirect, url_for app = Flask(__name__) def init_db(): conn = sqlite3.connect('database.db') c = conn.cursor() c.execute('''CREATE TABLE IF NOT EXISTS users (id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL, email TEXT NOT NULL)''') conn.commit() conn.close() @app.route('/users') def users(): conn = sqlite3.connect('database.db') c = conn.cursor() c.execute('SELECT * FROM users') users = c.fetchall() conn.close() return render_template('users.html', users=users) @app.route('/add_user', methods=['POST']) def add_user(): name = request.form['name'] email = request.form['email'] conn = sqlite3.connect('database.db') c = conn.cursor() c.execute('INSERT INTO users (name, email) VALUES (?, ?)', (name, email)) conn.commit() conn.close() return redirect(url_for('users')) if __name__ == '__main__': init_db() app.run(debug=True)

Using SQLAlchemy (ORM)

For more complex applications, you might want to use SQLAlchemy:

from flask import Flask from flask_sqlalchemy import SQLAlchemy app = Flask(__name__) app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///test.db' db = SQLAlchemy(app) class User(db.Model): id = db.Column(db.Integer, primary_key=True) name = db.Column(db.String(80), nullable=False) email = db.Column(db.String(120), unique=True, nullable=False) def __repr__(self): return f'<User {self.name}>' # Create tables with app.app_context(): db.create_all()

Deploying Flask Applications

There are several options for deploying Flask applications.

Production WSGI Servers

  • Gunicorn: Popular WSGI server for UNIX systems
  • uWSGI: Full-featured WSGI server
  • Waitress: Pure Python WSGI server for Windows and UNIX

Cloud Platforms

  • Heroku: Easy deployment with git push
  • PythonAnywhere: Simple Python hosting
  • AWS Elastic Beanstalk: Amazon's platform-as-a-service
  • Google App Engine: Google's serverless platform

Example with Gunicorn

# Install Gunicorn pip install gunicorn # Run your app with Gunicorn gunicorn app:app
